Code signing policy

Cada versión de Flobro se construye automáticamente a partir del código fuente público. Esta página describe quién puede cambiar ese código y cómo se firman las versiones.

Firma

Las versiones para macOS están firmadas y notarizadas con un Apple Developer ID, así que se abren sin aviso de Gatekeeper. Las versiones para Windows aún no están firmadas, por lo que Windows puede mostrar un aviso de SmartScreen la primera vez (elige Más información y luego Ejecutar de todas formas); la firma gratuita a través de SignPath Foundation está prevista en cuanto el proyecto cumpla los requisitos. Los binarios se generan con GitHub Actions directamente desde el repositorio; ninguna versión se construye en un equipo personal.

Roles del equipo

  • Committers y revisores: miembros de la organización flobro
  • Aprobadores: propietarios de la organización

Los cambios propuestos por personas ajenas al equipo solo se fusionan tras la revisión de un miembro del equipo. Cada solicitud de firma la aprueba manualmente un aprobador.

Privacidad

Flobro solo transmite las estadísticas anónimas de uso descritas en la política de privacidad, y solo cuando ese ajuste está activado. Se puede desactivar en la app y en la extensión.
